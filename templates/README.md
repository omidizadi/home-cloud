# Templates

This directory contains reference copies of the config files that the installer
generates. The installer itself embeds these via the step modules in
`homecloud/steps/`, so these files are for **documentation and manual recovery**
purposes only.

| File | Purpose | Generated by |
| :-- | :-- | :-- |
| `immich/docker-compose.yml.j2` | Immich stack (server + ML + redis + postgres) | `steps/immich.py` |
| `immich/env.j2` | Immich .env (secrets + config) | `steps/immich.py` |
| `homecloud-replug.sh` | SSD hot-replug recovery script | `steps/hardening.py` |
| `99-data.rules` | udev rule: trigger replug on SSD plug-in | `steps/hardening.py` |
| `data-replug.service` | systemd oneshot calling the replug script | `steps/hardening.py` |
| `homecloud-bot.service` | systemd unit for the Telegram bot | `steps/telegram_bot.py` |
| `docker-wait-for-ssd.conf` | systemd override making Docker wait for SSD | `steps/hardening.py` |

If you ever need to manually recreate a file (e.g. after a SD card failure),
refer to the corresponding step module for the authoritative version.
